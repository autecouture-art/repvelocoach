# チャピ式 VBT × AI トレーニングアプリ仕様書（Cursor用 完全版）

## 0. プロジェクト概要

本アプリは、**VBT（Velocity-Based Training）センサーの速度データ × AI解析**をベースに、

- その日の調子に合わせたトレーニング自動調整  
- 特殊セット（AMRAP / ドロップセット / スーパーセット）完全対応  
- **VBT機器なしのトレーニング（ノンVBT日）でも記録できる**  
- 種目別プロファイル（Vmax / 1vRM / LVP）自動更新  
- **リアルタイム累計ボリューム表示**  
- 各種PR（記録更新）の自動検出 & 通知  
- AIが褒めてモチベーションを上げてくれる  

という **完全自動VBTコーチアプリ** を目指すプロジェクト。

---

## 1. 主要機能

### 1.1 VBTモード（センサー使用）

- BLE接続（OVR、IMU等）
- Mean / Peak Velocity, ROM, Rep Duration を取得
- 外れ値（±2SD）除去
- Velocity Loss（VL）自動計算
- ウォームアップ速度比較 → **Δvで調子判定**
- トップシングル重量やバックオフ重量をAIが自動調整

### 1.2 ノンVBTモード（手入力）

以下を手入力で記録：

- 重量（または自重＋追加負荷）
- レップ数
- RPE（任意）
- テンポ (例: 2-0-1-0)
- セット種別（normal / amrap / drop / superset_A / superset_B）
- メモ

ノンVBTデータは **重量 × レップ × RPE** により簡易e1RM推定（精度は低め）

VBTとノンVBTを同じタイムライン上で表示。

### 1.3 ミックスモード対応

**各セットごとに「VBT計測」 or 「手入力」を切り替え可能。**

例：  
- ベンチのトップセットだけVBT  
- 補助種目はノンVBT  
→ 全て同一セッションに統合される。

### 1.4 種目プロファイル管理（LVP）

種目ごとに以下を保存：

- **Vmax（最軽重量での最大速度）**
- **1vRM（1RM時の平均速度）**
- **LVP直線モデル（load–velocity profile）**
- **VL挙動分類（急落 / 漸減 / 揺らぎ）**

用途：

- e1RM推定  
- 速度帯（パワー / 筋肥大 / 筋力）設定  
- 当日の調子判定  
- ドロップセットの重量提案に使用  

### 1.5 マシン重量ステップ自動認識

- ラットプル等のマシン写真を撮影
- 画像解析で重量ステップ（5kg/7kg刻み等）を抽出
- 種目プロファイルに保存 → 以後はプルダウン選択可能

### 1.6 特殊セット完全対応

#### ⭐ AMRAP
- VBT計測 → 失速ポイントをAIが解析  
- 手入力 → レップ × RPEから強度を推定  
- AIが「残りRIR推定」「次セット提案」を行う

#### ⭐ スーパーセット
- セット種別を `superset_A`, `superset_B` としてリンク保存  
- 心拍推定モデルで次セット開始タイミングを案内

#### ⭐ ドロップセット（重点機能）
ロジック：
1. 最後の有効レップ速度から現在の%1RMを逆算  
2. 目的別に次重量を自動設定  
   - 筋肥大 → −15〜25%  
   - 筋力 → −10〜15%  
3. ドロップ中の速度回復を監視し  
   - 「あと2レップ可能です」などリアルタイム表示  
4. 種目別にVL挙動を学習  
   - 急落型（チンニング）  
   - 漸減型（ベンチ/SQ）  
   - 揺らぎ型（デッドリフト）  

UI：
- 次重量へ自動切替  
- 目標速度帯を同時表示

## 2. リアルタイム累計ボリューム管理

計算式：
```
set_volume = load_kg × reps
session_volume = Σ set_volume
```

表示：
- 今日の累計ボリューム  
- 種目別ボリューム  
- フェーズ（筋肥大 / 筋力 / ピーク）基準で色表示  

AIコメント例：
> 「プレス系ボリュームは先週平均の78%。もう1セットは安全圏ですが、VLが高いため調整を推奨します。」

VL合計（50–60%）との相関も考慮。

## 3. PR（パーソナルレコード）検出 & 通知

検出するPR：
1. **e1RM-PR**  
2. **Speed-PR（同重量の最高速度更新）**  
3. **Set-PR（同重量×レップの最高速度）**  
4. **Volume-PR（種目別 or 全体ボリューム）**

通知：
- セット終了直後にポップアップ通知  
- セッション終了時に「今日のPRまとめ」表示  

褒めメッセージ例：
```
🎉 e1RM 更新！
ベンチプレス推定1RMが 112.5kg（+2.0kg）になりました。
動きが安定し、調子が非常に良いです！
```

```
⚡ Speed-PR！
スクワット120kgの平均速度が 0.52m/s（+0.04）に向上！
技術と力発揮が噛み合っています！
```

スパム防止：  
同種PRの即時通知はセッション内で最大1〜2回まで。

## 4. データ構造（統一仕様）

### レップ単位の記録フォーマット：

```json
{
  "session_id": "2025-12-09",
  "lift": "Bench Press",
  "set_index": 2,
  "rep_index": 1,

  "load_kg": 80.0,
  "device_type": "VBT | manual",

  "mean_velocity": 0.42,
  "peak_velocity": 0.73,
  "is_valid_rep": true,

  "rpe_set": 8.0,
  "set_type": "normal | amrap | drop | superset_A | superset_B",

  "notes": "右肩少し張り",
  "timestamp": "2025-12-09T18:23:00+09:00"
}
```

- VBTなし時は `mean_velocity: null`
- セット・種目単位に集計可能

## 5. AIロジックまとめ

### 5.1 e1RM推定
- VBT日 → LVP × 1vRM × 速度で高精度推定  
- ノンVBT日 → 重量 × レップ × RPEから簡易推定  

### 5.2 調子判定（Δv）
- 基準速度との差（±0.05m/s）で好調/不調  
- ノンVBT日は RPE比較から推定

### 5.3 トレーニング自動調整
- トップシングル重量 ±2.5〜5kg  
- バックオフ重量 ±2.5〜5%  
- セット数増減  
- VLが高い時は「中断提案」  
- ボリュームと疲労の両方からセッション最適化
